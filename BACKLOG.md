# Public Contribution Backlog

This backlog lists useful, independently reviewable work for external contributors. Items should become GitHub issues before implementation when coordination or acceptance criteria are needed.

## Documentation quality

- Audit every guide for a visible evidence-confidence statement.
- Standardise related-material links across achievement pages.
- Review historical pages for date precision and source status.
- Add a glossary for achievement, badge, tier, trigger, and profile visibility terms.

## Evidence research

- Collect dated observations for delayed achievement awards.
- Compare community-reported tier thresholds against current public profiles.
- Document account and repository visibility effects without exposing private data.
- Review whether retired badges remain visible on existing profiles.

## Accessibility and design

- Test keyboard navigation across all site routes.
- Review colour contrast and focus indicators.
- Add descriptive alternative text where images are introduced.
- Check tables and cards at narrow mobile widths.

## Tooling

- Extend link validation to generated Pages output.
- Add Markdown style checks.
- Add a Jekyll build validation workflow.
- Pin third-party actions to reviewed commit SHAs.

## Search and discovery

- Validate JSON-LD after deployment.
- Add breadcrumb structured data to guide pages.
- Review titles and descriptions for duplicated search snippets.
- Create a machine-readable achievement catalogue.

## Good first contributions

- Fix one confirmed broken internal link.
- Clarify one uncertainty label.
- Improve one page's heading hierarchy.
- Add one missing cross-reference.
- Correct one outdated or ambiguous sentence with evidence.

## Out of scope

The project does not accept empty commits, fabricated observations, fake engagement, deceptive attribution, mass automated activity, or changes whose primary purpose is inflating contribution counts.
