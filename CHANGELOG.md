# Changelog

All notable public changes to the GitHub Achievement Encyclopedia are recorded here.

The project uses semantic versioning for tagged documentation releases in accordance with [the release policy](RELEASES.md).

## Unreleased

No unreleased changes are currently recorded.

## v1.0.0 — 2026-07-19

### Added

- Nine standardised achievement guides covering seven active and two retired achievements.
- Evidence classifications, verification methodology, maintenance governance, and release controls.
- Repository-wide Markdown, catalogue, link, metadata, accessibility, and Jekyll validation.
- Verification-date freshness reporting and source resilience inventories.
- Desktop and mobile visual-regression baselines.
- Flagship README, GitHub Pages navigation, and accessible repository artwork.

### Changed

- Established the catalogue-linked guides as the canonical release scope.
- Repositioned the project as an evidence-led encyclopedia rather than an achievement-unlocking checklist.
- Adopted semantic versioning for formal tagged releases.

### Corrected

- Resolved repository-wide Markdown debt and affected internal routes.
- Completed missing verification-date metadata on active canonical guides.

### Verification

- Recorded the first formal verification baseline in [`docs/releases/v1.0.0.md`](docs/releases/v1.0.0.md).

## 2026-07-18

### Added

- Evidence-led reference pages for the principal GitHub profile achievements.
- GitHub Pages navigation, responsive cards, custom styling, sitemap, robots policy, and social-preview artwork.
- Correction and evidence issue forms.
- Pull-request template, accessibility checks, link checks, and public roadmap.
- Historical reference for retired and restricted badges.

### Changed

- Repositioned the repository as the GitHub Achievement Encyclopedia.
- Replaced the stock landing page with a structured public reference site.

### Notes

The encyclopedia distinguishes official documentation from dated observations and community reports.
