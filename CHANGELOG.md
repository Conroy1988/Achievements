# Changelog

All notable public changes to the GitHub Achievement Encyclopedia are recorded here.

The project uses semantic versioning for tagged documentation releases in accordance with [the release policy](RELEASES.md).

## Unreleased

### Added

- Claim-level evidence register covering thirteen canonical trigger, tier, and historical claims.
- Contradiction and dispute ledger with six explicit ambiguities and measurable resolution criteria.
- Evidence-coverage matrix with transparent confidence weights, dispute penalties, and unassigned-gap detection.
- Deterministic research priority board exposing every score component for the eight open contributor tasks.
- Official-document change-impact map connecting monitored GitHub pages to claims, disputes, achievements, and research work.
- Five supplemental machine-readable research-intelligence endpoints and a dedicated scheduled validation workflow.

### Changed

- Public navigation and API documentation now expose the complete research-intelligence layer.

## v1.2.0 — 2026-07-19

### Added

- Privacy-safe public evidence register with nine canonical evidence records and a static JSON endpoint.
- Weekly official GitHub documentation monitor with reviewed content fingerprints and affected-guide mapping.
- Achievement verification timelines with page-level last-reviewed dates and explicit history limitations.
- Contributor research hub containing eight bounded tasks, three good-first research opportunities, and a structured issue form.
- Research endpoints for evidence records, verification timelines, and the contributor queue.

### Changed

- Unified repository audit expanded from 12 to 17 controls, adding evidence, timeline, research-queue, official-baseline, and public-API contracts.
- Public API discovery expanded to identify the research infrastructure endpoints.
- All GitHub Actions dependencies upgraded where applicable and pinned to immutable commit SHAs.
- Repository navigation and audit documentation now expose the complete research workflow.

### Verification

- Recorded the research infrastructure release in [`docs/releases/v1.2.0.md`](docs/releases/v1.2.0.md).

## v1.1.0 — 2026-07-19

### Added

- Generated repository health dashboard and machine-readable operational status.
- Validated nine-record achievement dataset and JSON Schema contract.
- Accessible search and filtering across achievements and major project references.
- Single-command, 12-control repository audit with Markdown and JSON reporting.
- Static public API with discovery, aggregate, individual achievement, schema, and health endpoints.

### Changed

- Search matching now uses exact token and alias semantics with project-base-aware result links.
- Formal release verification now uses the unified repository audit and public API drift validation.
- Repository navigation now exposes health, audit, data, search, and API surfaces directly.

### Verification

- Recorded the feature and API release in [`docs/releases/v1.1.0.md`](docs/releases/v1.1.0.md).

## v1.0.0 — 2026-07-19

### Added

- Nine standardised achievement guides covering seven active and two retired achievements.
- Evidence classifications, verification methodology, maintenance governance, and release controls.
- Repository-wide Markdown, catalogue, link, metadata, accessibility, and Jekyll validation.
- Verification-date freshness reporting and source resilience inventories.
- Desktop and mobile visual-regression baselines.
- Flagship README, GitHub Pages navigation, and accessible repository artwork.

### Changed

- Established the catalogue-linked guides as the canonical release scope.
- Repositioned the project as an evidence-led encyclopedia rather than an achievement-unlocking checklist.
- Adopted semantic versioning for formal tagged releases.

### Corrected

- Resolved repository-wide Markdown debt and affected internal routes.
- Completed missing verification-date metadata on active canonical guides.

### Verification

- Recorded the first formal verification baseline in [`docs/releases/v1.0.0.md`](docs/releases/v1.0.0.md).
