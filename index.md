A community-maintained reference for GitHub profile achievements—built around reproducible evidence, clear uncertainty labels, and responsible participation.

> GitHub describes Achievements as a public-preview feature. Rules and thresholds can change, so this encyclopedia separates official documentation from community-observed behaviour.

## Evidence labels

| Label | Meaning |
|---|---|
| Official | Directly supported by current GitHub documentation or an official announcement |
| Reproduced | Independently observed with dated, reviewable evidence |
| Community consensus | Consistently reported, but not formally specified by GitHub |
| Unconfirmed | Plausible, incomplete, or awaiting independent reproduction |

## Project principles

1. **Evidence before certainty.** Unsupported claims are labelled rather than presented as fact.
2. **Useful activity only.** Empty commits, fake engagement, fabricated reviews, and misleading attribution are rejected.
3. **Privacy by default.** Evidence should exclude billing data, private repository content, tokens, and personal information.
4. **Corrections remain visible.** Material changes should explain what changed and why.
