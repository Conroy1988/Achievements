# Verification Calendar

This calendar converts the maintenance policy into repeatable review work. Dates are recorded in pull requests or audit reports; reviews must not rely on memory alone.

## Monthly review

Complete during the first full week of each month.

- Review open correction and evidence issues.
- Review failed or cancelled workflows.
- Review dependency and security notices.
- Confirm the public homepage and achievement index build successfully.
- Triage guides whose primary references are unavailable.

## Quarterly guide rotation

Review one guide group each quarter so every active guide receives at least one focused annual check.

| Quarter | Guide group |
|---|---|
| Q1 | Pull Shark, Pair Extraordinaire, and YOLO |
| Q2 | Galaxy Brain and Quickdraw |
| Q3 | Starstruck and Public Sponsor |
| Q4 | Arctic Code Vault Contributor, Mars 2020 Contributor, and cross-guide navigation |

Each quarterly review checks trigger wording, eligibility, progression claims, award timing, evidence labels, references, troubleshooting, and the last-verified date.

## Annual repository review

Complete once per calendar year.

1. Run the full link baseline.
2. Build the production Jekyll site.
3. Review keyboard focus and narrow-viewport navigation.
4. Validate structured data, canonical metadata, social image, sitemap, and robots policy.
5. Compare the achievement index with the navigation hub.
6. Review governance, contribution, security, maintenance, and release policies.
7. Review every active guide's evidence status and verification date.
8. Publish a dated annual verification report.

## Event-triggered review

Start a focused review when GitHub announces or visibly changes:

- profile achievements or badges;
- pull-request merge or review attribution;
- Discussions accepted answers;
- repository stars or ownership behaviour;
- GitHub Sponsors visibility;
- contribution attribution or co-author handling;
- GitHub Pages, Jekyll, or metadata behaviour relevant to the site.

## Stale-review rule

A guide that has not received a complete review within twelve months must display its existing evidence cautiously until reverified. Unavailable sources or conflicting observations require an explicit issue and may require a confidence downgrade.

## Review record

Each completed review records:

- date;
- reviewer;
- files and product surfaces checked;
- sources consulted;
- changes made or no-change conclusion;
- validation results; and
- unresolved limitations.
