## v1.3.0 — Evidence operations

Released **19 July 2026**.

## Added

- Claim-level evidence ownership with zero unassigned weak claims.
- Twelve-task contributor research queue and deterministic priority ranking.
- Five privacy-safe reproduction protocols for the highest-value active research areas.
- Automated evidence issue triage with prohibited-data screening.
- Draft evidence-record and draft pull-request creation after passing screening.
- Read-only public-signal profile auditor.
- Public research command centre.
- Expanded API discovery covering twenty-five public JSON files.
- Dedicated evidence-operations validation workflow.

## Verification

- Research-intelligence generator validates thirteen claims and six contradiction records.
- Evidence-operations generator validates twelve tasks, five protocols, nine auditor rules, and the submission schema.
- Weak claims without assigned research are rejected.
- Generated API drift is checked.
- Evidence submissions remain subject to mandatory human review.
- Repository operational health was verified at **100/100 — Excellent** before release preparation.

## Known limitations

- Evidence coverage remains **44.2/100** until independent observations strengthen the underlying claims.
- The profile auditor analyses public signals and cannot detect badge visibility.
- Unauthenticated GitHub API requests are rate-limited and omit private activity.
- Automated privacy screening reduces risk but cannot guarantee that user-submitted text is safe.
- Draft evidence pull requests do not alter canonical evidence until a maintainer reviews and merges them.
