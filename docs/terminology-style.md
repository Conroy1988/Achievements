# Terminology and Style

Use consistent language throughout the reference.

- Capitalise official achievement names exactly as GitHub displays them.
- Use default, bronze, silver, and gold for tiers.
- Say community-observed threshold when GitHub does not formally document a number.
- Distinguish opened, closed, and merged pull requests.
- Distinguish a commit author from a co-author trailer.
- Use exact dates for verification records.
- Avoid claims such as guaranteed or instant unless official documentation supports them.

Clear terminology prevents procedural differences from being hidden by casual wording.
