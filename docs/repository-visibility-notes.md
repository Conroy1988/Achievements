# Repository Visibility Notes

Achievement tests should always record whether the relevant repository was public, private, or internal.

Visibility can affect what evidence other maintainers can inspect and may affect undocumented product behaviour. Do not assume that a result reproduced in a public repository is identical in a private repository.

When private activity is involved, publish only the minimum metadata needed to describe the test. Never expose private source code, collaborator identities, or repository names solely to prove an achievement event.
