# Evidence strength levels

Use consistent evidence grades when documenting GitHub achievement behaviour.

## Confirmed

Supported by current official GitHub documentation or directly observable platform output.

## Reproduced

Repeated successfully under recorded conditions by more than one test.

## Observed

Seen in a credible example but not yet independently reproduced.

## Community reported

Reported by users or third-party references without sufficient direct verification.

## Unverified

Plausible but unsupported. Do not present as fact.

Always record the verification date and link the strongest available source.
