# Reproducibility checklist

Before treating an observed achievement trigger as reproducible, record:

- account and repository visibility conditions;
- the precise qualifying action;
- event timestamps and time zone;
- branch, pull request, issue, or discussion identifiers;
- review and merge conditions;
- observed processing delay;
- whether the result was independently repeated;
- any platform settings that may affect attribution.

A failed reproduction should be retained when it reveals a meaningful constraint.
