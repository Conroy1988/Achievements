# Profile Processing Delays

A qualifying GitHub event and the appearance of an achievement badge may not be simultaneous.

When verifying a result:

- preserve the public link and timestamp for the qualifying event;
- record the account's prior achievement tier;
- check the profile periodically without repeating the event unnecessarily;
- distinguish a delayed badge from a rejected qualification;
- avoid asserting a universal processing time from one observation.

Reports should state the observed delay, if known, and the date the profile was last checked.
