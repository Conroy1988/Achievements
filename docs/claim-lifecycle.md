# Achievement claim lifecycle

Achievement claims should move through defined states.

1. **Submitted** — a new claim or correction is recorded.
2. **Triaged** — scope, evidence, and affected documentation are identified.
3. **Under review** — sources and reproducibility are checked.
4. **Accepted** — evidence meets the publication standard.
5. **Disputed** — reliable sources conflict or reproduction fails.
6. **Superseded** — newer platform behaviour replaces the earlier conclusion.
7. **Archived** — the claim remains historically useful but is no longer current.

State changes should include a date and concise rationale.
