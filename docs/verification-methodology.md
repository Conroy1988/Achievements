# Verification Methodology

Use this process when confirming an achievement condition or tier.

## Test design

- Define the exact qualifying event before acting.
- Record the account, repository visibility, timestamps, and relevant repository settings.
- Change one material variable at a time.
- Avoid disposable accounts, fabricated collaboration, spam, or misleading activity.

## Observation

After the event, check the profile achievement panel and record when the badge appears. A delay does not necessarily mean the event failed because profile processing may be asynchronous.

## Reporting

Document the action, observed result, date, uncertainty, and whether the result agrees with official or community documentation. Never present a single test as proof of a universal rule.
