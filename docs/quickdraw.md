# Quickdraw

## Summary

Quickdraw is an active, non-tiered GitHub profile achievement associated with closing an issue or pull request shortly after opening it.

## Trigger

The reported qualifying event is opening an issue or pull request and closing it within a short interval. GitHub does not publish the precise timing window in its general profile documentation.

## Progression or tiers

Quickdraw is generally observed as a single-level achievement with no published tier progression.

## Eligibility conditions

- The same item must be opened and then closed.
- The event must occur on GitHub as an issue or pull request.
- Reopening, transferring, converting, or deleting an item can complicate later verification.
- Repository rules and permissions still apply.

## Award timing

The item can show as closed immediately while the profile achievement appears later. GitHub does not guarantee a fixed processing interval.

## Verification

1. Open the issue or pull request.
2. Confirm the creation and closure timestamps.
3. Confirm the account attribution.
4. Preserve the public URL where appropriate.
5. Recheck the profile after allowing for processing delay.

## Evidence status

- GitHub officially documents opening and closing issues and pull requests.
- The association with a rapid close is strongly observed.
- The exact timing window remains community-reported or unknown.

## Known limitations and edge cases

- Timestamps displayed in the interface may be rounded.
- Automated closure, duplicate handling, or moderation can obscure the user's role.
- Private repository evidence may not be publicly reviewable.
- Closing useful work prematurely solely for the achievement is not responsible participation.

## Troubleshooting

- Confirm the item is an issue or pull request rather than a Discussion.
- Check both timestamps directly.
- Confirm it was closed rather than merely converted or transferred.
- Allow time for profile processing.
- Do not assume a single failed attempt disproves the reported trigger.

## Responsible participation

Use a legitimate short-lived issue or pull request, such as immediately correcting an accidental duplicate or withdrawing an invalid proposal. Do not create disruptive items in third-party repositories.

## History

Quickdraw was introduced with GitHub profile achievements. Complete trigger and timing details have not been publicly documented by GitHub.
