# Evidence Records

Evidence supporting achievement claims should be stored or linked in a form that another maintainer can inspect.

A record should identify the achievement, tier, qualifying event, date, repository visibility, relevant settings, observation delay, and evidence classification. Public GitHub links are preferred. Screenshots should be used only when the product state cannot be represented by a durable public event.

Do not upload secrets, billing details, private repository content, or unrelated personal information. When evidence becomes stale or inaccessible, mark the associated claim for re-verification rather than treating the old conclusion as permanent.
