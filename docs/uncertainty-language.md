# Uncertainty language

Use wording that reflects the strength of available evidence.

- **GitHub documents** — use when supported by current official documentation.
- **Testing confirms** — use after a recorded, repeatable test.
- **Current observations indicate** — use for consistent but incompletely documented behaviour.
- **Community reports suggest** — use for credible third-party reports awaiting reproduction.
- **Unknown** — use when evidence is insufficient.

Avoid absolute claims such as “always,” “never,” or “guaranteed” unless the source explicitly supports them.
