# Change Classification

Classify pull requests so maintainers can assess impact consistently.

- **Correction** — fixes an inaccurate claim or threshold.
- **Verification** — records a new controlled reproduction.
- **Documentation** — improves clarity without changing a substantive claim.
- **Maintenance** — updates links, dates, formatting, or project infrastructure.
- **Policy** — changes contribution, conduct, privacy, or evidence standards.
- **Historical** — adds context about retired or unavailable achievements.

Pull-request descriptions should state the classification and identify any claims that require re-verification.
