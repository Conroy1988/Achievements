# Documentation update review workflow

Use this workflow for material changes to achievement guidance.

1. Identify the affected claim, guide, or threshold.
2. Check current official documentation first.
3. Compare independent evidence and dated observations.
4. Record uncertainty where behaviour is not fully documented.
5. Review links, terminology, and accessibility.
6. Open a focused pull request with a concise rationale.
7. Merge only after the change is internally consistent.
8. Record notable changes in release notes when appropriate.

Avoid combining unrelated corrections in one review.
