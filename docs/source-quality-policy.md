# Source Quality Policy

This project distinguishes between official GitHub documentation, directly reproduced behaviour, and community-observed thresholds.

## Evidence levels

1. **Official** — documented by GitHub in product documentation or changelogs.
2. **Reproduced** — observed through a documented test with dates and qualifying activity.
3. **Community observed** — consistently reported by independent references but not formally documented by GitHub.
4. **Unverified** — plausible but lacking sufficient evidence.

## Citation requirements

Every numerical threshold or behavioural claim should include a source, reproduction note, or explicit uncertainty label. Screenshots should include the date and redact private information.

## Maintenance

Achievement behaviour may change without notice. Contributors should re-check time-sensitive claims before updating a guide and record the verification date.
