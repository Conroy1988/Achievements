# Verification summary format

Use this structure for concise verification conclusions:

- **Claim:** achievement behaviour being assessed.
- **Status:** unverified, observed, reproduced, confirmed, disputed, or retired.
- **Result:** what occurred.
- **Conditions:** account, repository, visibility, and event context.
- **Evidence:** links and dated records.
- **Limitations:** unresolved uncertainty or missing coverage.
- **Verified:** date and reviewer.

The summary should separate facts from inference and avoid claiming more certainty than the evidence supports.
