# Status Labels

Use consistent labels when documenting achievement information.

- **Official** — supported by current GitHub documentation or product text.
- **Reproduced** — confirmed through a dated, controlled test.
- **Community observed** — supported by multiple independent reports but not guaranteed by GitHub.
- **Needs verification** — evidence is incomplete, contradictory, or stale.
- **Retired** — no longer available to newly qualifying users.
- **Unavailable** — announced or visible in assets but not currently earnable.

Every achievement guide should identify its status and last verification date near the top of the document.
