# Review Cadence

Achievement behaviour and thresholds can change without a formal announcement. This project uses the following maintenance cadence:

- Review official GitHub documentation and changelogs quarterly.
- Re-check community-observed thresholds every six months.
- Reproduce disputed behaviour when a controlled test is practical.
- Review retired and unavailable achievements annually.
- Triage correction reports as they arrive.

Each review should record its date, sources checked, findings, and any unresolved uncertainty. A review that finds no changes is still useful and may be recorded in a pull request or maintenance issue.
