# Starstruck

## Summary

Starstruck is an active, tiered GitHub profile achievement associated with stars received by a repository owned by the user.

## Trigger

A qualifying signal is a GitHub user starring a repository associated with the account seeking the achievement. Stars indicate interest; they are not a review or guarantee of project quality.

## Progression and tiers

GitHub does not publish the complete tier table in its general profile documentation. The following values are community-reported:

| Level | Repository stars | Evidence status |
|---|---:|---|
| Base | 16 | Community-reported |
| Bronze | 128 | Community-reported |
| Silver | 512 | Community-reported |
| Gold | 4,096 | Community-reported |

## Eligibility conditions

- The repository must be attributable to the account seeking credit.
- Stars must represent ordinary GitHub star events.
- Fork ownership, transferred repositories, deleted repositories, and organisation ownership can complicate attribution.
- GitHub does not publicly guarantee how every ownership transition affects historical progress.

## Award timing

Repository star counts update quickly, but profile-achievement processing may lag. A repository can visibly reach a reported threshold before the badge or tier refreshes.

## Verification

1. Open the repository page.
2. Confirm the visible star count.
3. Confirm the repository owner and account attribution.
4. Record the repository URL and verification date.
5. Recheck the profile after allowing for processing delay.

## Evidence status

- GitHub officially documents repository stars and starring controls.
- The association between qualifying repository stars and Starstruck is strongly observed.
- Exact tier thresholds remain community-reported.
- Ownership-transfer and aggregation behaviour is not fully documented.

## Known limitations and edge cases

- Star counts can decrease when users remove stars or accounts are removed.
- A screenshot captures only a point-in-time count.
- Organisation-owned repositories may not behave like personally owned repositories.
- Repository transfers, visibility changes, deletion, or restoration can affect verification.

## Troubleshooting

- Confirm the repository is owned by or correctly attributed to the account.
- Confirm the visible count has reached the reported threshold.
- Allow time for profile processing.
- Check whether stars were removed after the threshold was reached.
- Treat unexpected ownership or transfer behaviour as unresolved unless reproducible evidence is available.

## Responsible participation

Build useful software or documentation, explain it clearly, and share it with relevant communities. Stars should reflect independent interest in the project rather than coordinated activity created solely to alter the count.

## History

Starstruck was introduced with GitHub profile achievements. GitHub may change qualification or display behaviour without publishing a complete threshold history.
