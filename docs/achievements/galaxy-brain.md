# Galaxy Brain

## Summary

Galaxy Brain is an active, tiered GitHub profile achievement associated with helpful answers in GitHub Discussions that are marked as accepted.

## Trigger

A qualifying event is an answer authored by the user in a repository Discussion and subsequently marked as the accepted answer. Ordinary comments, issue replies, pull-request reviews, and unaccepted Discussion replies do not establish the same event.

## Progression and tiers

GitHub does not publish the complete tier table in its general profile documentation. The following thresholds are therefore recorded as community-reported rather than guaranteed:

| Level | Accepted answers | Evidence status |
|---|---:|---|
| Base | 2 | Community-reported |
| Bronze | 8 | Community-reported |
| Silver | 16 | Community-reported |
| Gold | 32 | Community-reported |

## Eligibility conditions

- The repository must have GitHub Discussions enabled.
- The Discussion category must support marking an answer.
- The response must be posted as an answer by the account seeking credit.
- A repository maintainer or the Discussion author must mark it as accepted.
- Deleted, unaccepted, or non-answer contributions may not qualify.

## Award timing

Profile processing is not necessarily immediate. A qualifying accepted answer can appear in the Discussion before the achievement or tier is refreshed on the profile.

## Verification

1. Open the relevant Discussion.
2. Confirm the response is attributed to the correct account.
3. Confirm GitHub displays it as the accepted answer.
4. Record the Discussion URL and acceptance date without exposing private content.
5. Recheck the profile achievement after allowing for processing delay.

## Evidence status

- GitHub officially documents Discussions, answer-enabled categories, and accepted-answer controls.
- The relationship between accepted answers and Galaxy Brain is strongly observed.
- Exact numerical tier thresholds remain community-reported.
- No fixed award-processing interval is officially guaranteed.

## Known limitations and edge cases

- Not every Discussion category supports answers.
- A useful reply can remain unaccepted and therefore may not count.
- Removing acceptance or deleting content can complicate later verification.
- Private or access-restricted Discussions are unsuitable as public evidence.
- Screenshots prove visible state only; they do not independently establish the complete threshold model.

## Troubleshooting

- Confirm the contribution is in GitHub Discussions rather than Issues.
- Confirm the category supports accepted answers.
- Confirm the answer is visibly marked as accepted.
- Confirm the correct GitHub account authored it.
- Allow time for profile processing.
- Treat a missing higher tier as unresolved evidence rather than proof that the reported threshold is exact.

## Responsible participation

Answer real questions with accurate, actionable information. Include reproducible steps and relevant documentation where useful. Do not post superficial replies, pressure authors to accept answers, or coordinate artificial Discussions solely to obtain the badge.

## History

Galaxy Brain was introduced as part of GitHub's profile-achievement system. GitHub may change qualification logic or presentation without publishing a complete threshold changelog.
