# Make your first contribution

## Good starter changes

- Correct a typo or unclear sentence.
- Repair an internal documentation link.
- Add a missing source date.
- Improve alt text, heading order, or mobile readability.
- Clarify whether a claim is official, reproduced, community consensus, or unconfirmed.

## Workflow

1. Select an open starter issue or identify one focused improvement.
2. Comment on the issue when coordination would prevent duplicated work.
3. Fork the repository and create a clearly named branch.
4. Make the smallest complete change that resolves the issue.
5. Run the repository link validator when links changed.
6. Open a pull request and complete every relevant checklist item.
7. Respond to review with focused follow-up commits.

## Evidence contributions

Do not manufacture activity to test an achievement. Submit observations arising from genuine GitHub use, and remove private data before sharing screenshots or logs.

## Review expectations

A maintainer may request stronger sourcing, clearer uncertainty wording, reduced scope, accessibility corrections, or privacy redaction. This is normal editorial review rather than rejection of the contributor.
