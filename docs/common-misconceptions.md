# Common Misconceptions

## A merged commit is always equivalent to a merged pull request

Pull Shark is associated with pull requests opened by the user and subsequently merged. Direct commits to the default branch are not the same event.

## Repository owners can award achievements manually

Achievements are processed by GitHub. Repository maintainers cannot directly grant profile badges.

## Every threshold is officially guaranteed

Many tier counts are community observed rather than formally documented. They should be treated as current evidence, not permanent API contracts.

## Rapid activity guarantees immediate display

Profile processing can be delayed. Preserve links to qualifying events and allow for asynchronous processing.

## More activity is always better

Spam, fabricated collaboration, and meaningless changes reduce repository quality and are not recommended by this project.
