# Link Maintenance

Reliable references are essential for an evidence-led project.

- Prefer stable GitHub documentation and changelog URLs.
- Record an access or verification date for time-sensitive sources.
- Replace broken links with archived or successor sources where possible.
- Do not silently change a source when the replacement supports a materially different claim.
- Use descriptive link text and avoid tracking parameters.
- Flag references that require authentication or access to private content.

A link correction should also confirm that the surrounding claim remains supported.
