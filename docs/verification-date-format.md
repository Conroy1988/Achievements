# Verification Date Format

Use ISO 8601 calendar dates in verification metadata: `YYYY-MM-DD`.

Where timing affects the result, also include the local time, time zone, and UTC equivalent. Avoid relative descriptions such as today or last week because they become ambiguous after publication.

Each guide should state:

- last verified date;
- evidence classification;
- verifier or pull request;
- any scheduled re-check date.

Exact dates make it easier to compare reports and identify stale achievement information.
