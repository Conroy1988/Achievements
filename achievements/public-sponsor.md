# Public Sponsor

**Status:** Earnable  
**Tiered:** No  
**Evidence level:** The underlying GitHub Sponsors programme is official; the achievement trigger is described consistently by community references.

## Trigger

Public Sponsor recognises a publicly visible sponsorship made through GitHub Sponsors.

## Expected conditions

- Sponsor an eligible developer or organisation through GitHub Sponsors.
- Use a sponsorship tier that creates an active sponsorship.
- Keep the sponsorship publicly visible where the selected privacy controls allow it.
- Ensure Achievements are enabled on your profile.

## Privacy considerations

Sponsorship visibility and achievement visibility are separate profile controls. Review the displayed information before completing a sponsorship, particularly when using a personal account or payment method.

## What may prevent the badge

- Sponsoring privately or anonymously.
- A payment that has not completed.
- A cancelled or failed sponsorship.
- Profile achievements being hidden.
- GitHub processing delay.

## Responsible use

Choose a maintainer or project whose work you genuinely value. Sponsorship is financial support, not merely an achievement transaction, and may recur depending on the selected tier.

## Verification

Check the Sponsors section of your profile and the sponsorship's visibility status. Avoid publishing payment details, billing information, or private receipts as evidence.

## Important account limitation

GitHub's official profile documentation states that private profiles cannot receive sponsorships. Account and regional eligibility for GitHub Sponsors can also change, so consult the current GitHub Sponsors terms before paying.
