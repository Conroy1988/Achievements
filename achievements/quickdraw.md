# Quickdraw

**Status:** Earnable  
**Tiered:** No  
**Evidence level:** Strong community consensus; GitHub does not publish the exact rule in its general documentation.

## Trigger

Quickdraw is awarded after an issue or pull request is closed within approximately five minutes of being opened.

## What normally qualifies

- Create a legitimate issue or pull request.
- Resolve or determine it is unnecessary quickly.
- Close it within the reported timing window.
- Keep the activity attributable to the account earning the achievement.

## What does not help

- Closing an old issue quickly after discovering it.
- Reopening and reclosing an existing item.
- Empty or intentionally misleading reports created only to manufacture activity.

## Timing notes

The five-minute threshold is community reported rather than formally documented by GitHub. Use the timestamps on the issue or pull request as the primary evidence.

## Responsible route

A practical legitimate case is a duplicated issue, an immediately superseded pull request, or a test repository used to learn issue management with a clear explanation.

## Troubleshooting

Achievement processing may not be immediate. Confirm that achievements are visible in profile settings and allow GitHub time to process the event.

## Confidence statement

The trigger is consistently reproduced across long-running community references, but GitHub states that Achievements remain a public-preview feature and may change.
