# GitHub Achievement Encyclopedia roadmap

This roadmap describes the next useful improvements to the public reference. It is not a promise that GitHub's undocumented achievement rules will remain unchanged.

## Current foundation

- Evidence-led achievement index
- Individual guides for currently documented achievements
- Community-reported tier tables with explicit caveats
- Responsive GitHub Pages website
- Contribution and evidence guidance
- Retired and historical achievement reference
- Search metadata, sitemap support, and social-preview branding

## Near term

### Quality assurance

- Run automated internal and external link checks
- Audit representative pages against WCAG 2.1 AA expectations
- Resolve broken anchors, stale references, and mobile layout defects
- Confirm sitemap and crawler files after each material site change

### Community workflow

- Maintain structured forms for corrections and evidence submissions
- Label unsupported claims as unconfirmed until independently reproduced
- Improve contributor onboarding and review criteria
- Create good-first-issue opportunities for accessibility, wording, and source maintenance

### Content expansion

- Add dated source notes to every achievement guide
- Document award-delay observations without presenting a fixed guarantee
- Expand the historical timeline when reliable evidence is available
- Add a glossary for GitHub Discussions, co-authorship, sponsorship visibility, and achievement tiers

## Medium term

- Add a searchable achievement directory
- Add page-level last-reviewed dates
- Introduce structured data for article and FAQ pages
- Build a public evidence register with privacy-safe references
- Add translations only when maintainers can keep them synchronized
- Publish versioned research summaries for material rule changes

## Long term

- Establish a contributor review group for disputed findings
- Track changes in GitHub's official achievement documentation
- Maintain reproducible test protocols for undocumented behaviour
- Become a durable reference cited by community discussions and technical guides

## Out of scope

The project will not accept:

- Empty or meaningless activity created solely to increase counts
- Fake stars, reviews, sponsorships, identities, or engagement
- Spam against unrelated repositories
- Private data, credentials, billing details, or unredacted personal information
- Claims presented as official without an official source

## How priorities are chosen

Work is prioritised by reader impact, evidence quality, accessibility, maintainability, and the likelihood that inaccurate information could mislead contributors.
