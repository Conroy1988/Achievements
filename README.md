# GitHub Achievement Encyclopedia

A community-maintained reference for GitHub profile achievements: what they mean, how they are earned, which thresholds are confirmed, and where uncertainty remains.

> GitHub does not publish complete rules for every achievement or tier. This project separates official documentation, reproducible testing, dated observations, and community reports so readers can judge the strength of each claim.

## What this project covers

The encyclopedia documents active, tiered, retired, and limited-availability GitHub achievements. Each achievement guide aims to include:

- the visible achievement name and description;
- the qualifying action or event;
- known or reported tier thresholds;
- evidence strength and verification date;
- important exclusions, delays, and attribution rules;
- a legitimate route to qualification without spam or deceptive activity.

## Project principles

1. **Evidence before certainty.** Community observations are labelled as such.
2. **No artificial engagement.** The project does not promote spam, fake contributions, or misleading activity.
3. **Dates matter.** GitHub behaviour can change, so material claims should include a verification date.
4. **Corrections remain visible.** Disputed or superseded claims are corrected transparently rather than silently rewritten.
5. **Contributions should improve the reference.** Pull requests must add, correct, verify, organise, or maintain substantive content.

## Current status

The repository already contains a large evidence and methodology library. The next phase is converting that foundation into concise, discoverable achievement pages and a public GitHub Pages site.
